### Anti-patterns to avoid

- **Anti-Pattern 2 (reading all upstream artifacts):** Stage 2 reads ONLY the stage-1 handoff bundle, not the raw `design/research/` directory. The handoff bundle was built specifically to contain what Stage 2 needs.
- **FID-02 (color/typography in IA):** Gate-stage-2.mjs must reject sitemaps with `color` or `font` fields on any node. This is enforced at the script level, not by LLM discipline. Do NOT add color/typography fields to the sitemap JSON schema — sitemap.v1.json does not permit them.
- **D-40 evidence grade:** Pass-grade for Stage 2 is `proto` (no tree-test data). Do NOT return `evidence:'validated'` from gate-stage-2.mjs in v2.0a. `evidence:'validated'` requires tree-test results from the v2.1 `ia/tree-test-design` atom.
- **Stage 3/4 references:** Do NOT include references to Excalidraw, XState, stateDiagram-v2, or Stage 3/4 workflow steps in any SKILL.md in this plan. This plan covers Stage 2 only.
