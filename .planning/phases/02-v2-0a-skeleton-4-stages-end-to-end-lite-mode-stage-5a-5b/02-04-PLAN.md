**Constraint: No Stage 3 or Stage 4 references.** This plan does NOT import, reference, or mention Excalidraw, XState, stateDiagram-v2, or any Phase 3/4 artifact. The systematize-lite workflow operates entirely on Stage 5a tokens output. The DESIGN.md is emitted from tokens data alone.

---

### Anti-patterns to avoid

- **Frost ≥3× gate (Phase 3 item):** Do NOT implement occurrence-count enforcement in gate-stage-5b.mjs. Record the count in the gate findings as informational; attach the note "Frost ≥3× recurrence not yet verified (requires Phase 3 Stage 4 artifacts)" per D-44. The gate returns pass_with_warnings regardless of count as long as count ≥ 1.
- **evidence:validated on DESIGN.md:** The DESIGN.md must carry evidence:INFERRED in its $extensions.design-os frontmatter. If it carries evidence:validated, the frontmatter validator (extended in Plan 02-01) will reject it. The gate must enforce this.
- **Scope creep into full Stage 5b:** Do NOT add component scaffold generation, Storybook integration, or design system documentation generation. The v2.0a systematize-lite scope is: component-promotion count + DESIGN.md emit from tokens data. Everything else is Phase 3.
