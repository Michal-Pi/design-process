<objective>
Build the gate-runner machinery, the handoff-bundle pipeline, the manifest.lock hash chain, and the bundle-sufficiency eval harness. The gate-runner base supports `not_runnable` as a terminal state from day one (codex §16 BLOCKER, GATE-07); the stage-5a gate ships a hardcoded refusal path so Phase 2's lite mode is honest. Purpose: Without this plan, Phase 2's stage workflows have nowhere to plug their gates and no context-window survival mechanism between stages. The base class supporting `not_runnable` from day one is non-negotiable per codex §16 BLOCKER.

Output: 1 base gate-runner + 6 per-stage gate skeletons + 4 stage-gate checklists (stage-1, 2, 5a, 5b per D-25; stages 3/4 ship in Phase 3) + handoff-bundle script + manifest.lock appender + 5 bundle-sufficiency fixtures + the structural eval runner + comprehensive tests.
</objective>
