<objective>
Establish the Zod-first single source of truth for design-os's 6 canonical artifact types, the Draft 2020-12 JSON Schema emit pipeline, the ajv 8 runtime validator, the schema discovery manifest, the schema migration tooling, and the design-md-validate version-pinning scaffold. Every downstream plan (gates, handoff bundle, governance, preview) consumes these schemas, so this plan is Wave 1 with no dependencies.

Purpose: Lock the data contracts before any consumer is written. Without frozen versioned schemas in `schemas/dist/`, every other plan is building against unfrozen ground — exactly the rework-killer the v1.5 4-week budget exists to prevent. The `zod-to-json-schema` package named in D-01 was deprecated in November 2025; this plan implements D-01's intent using Zod 4's built-in `z.toJSONSchema()` (Draft 2020-12 by default) and documents the substitution.

Output: Six versioned JSON Schemas under `schemas/dist/`, a schema discovery manifest, an ajv runtime validator, the `design-os migrate` CLI scaffold, the frontmatter validator, and `design-md-validate.mjs` with FORMAT-07 version pinning. Tests under `tests/schemas/` validate every fixture round-trip and prove migration correctness.
</objective>

Frontmatter common (shared by every canonical artifact):
- artifact: string literal per artifact type
- stage: enum '0' | '1' | '2' | '3' | '4' | '5a' | '5b' | 'cross-stage'
- schemaVersion: integer ≥ 1
- sourceHash: string matching ^sha256:[0-9a-f]{64}$
- generated: ISO 8601 datetime
- provenance: enum 'generated' | 'validated' | 'inferred' | 'missing'
- worstProvenance: optional enum same values (RED-04 carrier; downstream plans rely on this)
- owner: string (email recommended but not enforced beyond non-empty)
- lastReviewedAt: ISO 8601 datetime

GateResult discriminated union (Plan 02 imports this; Plan 03 imports this; Plan 05 imports this):
- { kind: 'pass', evidence: 'validated' | 'proto' | 'inferred', findings: Finding[] }
- { kind: 'pass_with_warnings', evidence: 'validated' | 'proto' | 'inferred', findings: Finding[], warnings: string[] }
- { kind: 'failed_after_repair', reason: string, findings: Finding[] }
- { kind: 'user_overridden', reason: string, overrideBanner: string, findings: Finding[] }
- { kind: 'not_runnable', reason: string }   ← GATE-07 day-one terminal state

HandoffBundleV1 frontmatter (Plan 02 imports this):
- artifact: 'handoff-bundle'
- schemaVersion: 1
- stage: string like "1 → 2" or "5a → 5b"

- tokenCount: integer
- truncationWarning: nullable string
- bundle sections (required): goalAndScope, decisionsMade, openQuestions, artifactsInventory, pointersToVerify, provenanceWorstCase
- bundle sections (optional): risksSurfaced

Schema discovery manifest (`schemas/dist/index.json`):
