# Designing for the Digital Age — Kim Goodwin (Cooper)

<!-- complete-design reference: cooper-goodwin -->

## Purpose

Goodwin's goal-directed design process (from Cooper's methodology) defines how user research
translates into behavioral requirements and interaction specifications. Relevant for Stage 1
persona construction and Stage 4 interaction design.

## Citations

- Goodwin, K. (2009). *Designing for the Digital Age*. Wiley.
- Ch.3, p.63: "Goals are not tasks. A goal is what motivates the behavior; a task is just one
  way someone might try to achieve that goal." Design-os personas use goals, not task lists.
- Ch.5, p.117: Behavioral archetype vs. demographic segment — archetype clusters by behavioral
  patterns, not by age/gender/income. Persona JSON `provenance` field carries this distinction.
- Ch.6, p.145: Mental models precede interface design; Stage 2 IA should reflect the user's
  conceptual model, not the system's data model.
- Ch.9, p.221: Interaction framework — the basic structure of behavior across screens before
  any pixel-level decisions. Stage 4 IxD machines (XState / Mermaid) capture this.
- Ch.11, p.267: Design principles as criteria — each principle directly testable during gate
  review. Stage gate checklists cite design principles as `VALIDATED` evidence.

## How complete-design uses it

- Stage 1 (Research / Strategy plane): persona archetypes defined by behavioral goals per §Ch.3.
- Stage 2 (IA / Scope plane): mental-model alignment in sitemap review checklist.
- Stage 4 (IxD / Skeleton plane): interaction framework approach per §Ch.9.
