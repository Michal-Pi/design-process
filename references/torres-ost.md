# Continuous Discovery Habits — Teresa Torres (Opportunity Solution Tree)

<!-- design-os reference: torres-ost -->

## Purpose

Teresa Torres's Opportunity Solution Tree (OST) framework connects business outcomes to
user opportunities to solutions to experiments. Used in Stage 1 (Research / Strategy) to
structure discovery artifacts and tie user needs to product decisions.

## Citations

- Torres, T. (2021). *Continuous Discovery Habits*. Product Talk Press.
- Ch.1, p.8: "An outcome is a change in customer behavior that drives business results."
  Design-os Stage 0 PRD must include an outcome, not just a feature list.
- Ch.4, p.63: Opportunity mapping — "An opportunity represents a customer need, pain point,
  or desire." Stage 1 research synthesizes opportunities, not solutions.
- Ch.5, p.91: The Opportunity Solution Tree structure — opportunity nodes at top, solution
  nodes branch from opportunities, assumption nodes branch from solutions.
- Ch.6, p.111: "Good opportunities are customer-centric, not technology-centric." Stage 1
  gate check: each JTBD/OST node must reference a user behavior, not a feature.
- Ch.7, p.133: Assumption testing beats solution testing — Stage 1 gate requires ≥1
  assumption identified per major opportunity.

## How design-os uses it

- Stage 0 (Strategy): PRD outcome field aligned to Torres OST outcome definition.
- Stage 1 (Personas / Research): opportunity synthesis uses OST node vocabulary.
- Gate-1 checklist: each persona opportunity statement must be behavior-grounded (§Ch.4).
